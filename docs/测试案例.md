# Cloudflare Speed Test - 测试案例文档

> 参数组合、预期行为与自动化测试说明

---

## 1. 测试分类

| 类型 | 说明 | 运行方式 |
|------|------|----------|
| 单元测试 | Config 解析、CIDR 解析、调度模式、Hosts 解析等 | `dotnet test` |
| 参数组合 | 命令行参数与默认值 | 见下表 |
| 集成场景 | 需网络/文件的场景，见「集成场景」 | 手动或 CI |

---

## 2. 参数组合测试矩阵

### 2.1 IP 来源

| 用例 ID | 参数 | 预期 |
|---------|------|------|
| IP-01 | （无） | 使用 ip.txt，仅 IPv4 源 |
| IP-02 | `-ipv6` | 使用 ipv6.txt，仅 IPv6 源 |
| IP-03 | `-f custom.txt` | 加载 custom.txt，不按类型过滤 |
| IP-04 | `-f6 custom6.txt` | 需与 IP-02 同用，加载 custom6.txt |
| IP-05 | `-ip "104.16.0.0/13"` | 仅该 CIDR，IPv4 |
| IP-06 | `-ip "2606:4700::/32"` | 仅该 CIDR，IPv6 |
| IP-07 | `-ip "104.16.0.0/13,2606:4700::/32"` | 混合 v4+v6，均参与测速 |
| IP-08 | `-ipn 100` | 最多加载 100 个 IP |
| IP-09 | `-ipv6 -ip "104.16.0.0/13"` | 仍用 -ip，解析为 IPv4，正常测速 |

### 2.2 延迟测速模式

| 用例 ID | 参数 | 预期 |
|---------|------|------|
| PING-01 | （无） | ICMP Ping |
| PING-02 | `-tcping` | TCPing 443 |
| PING-03 | `-httping` | HTTPing |
| PING-04 | `-tcping -tl 150` | TCPing，延迟上限 150ms |
| PING-05 | `-n 100 -t 2` | 并发 100，每 IP 2 次 |
| PING-06 | `-httping -cfcolo "HKG,NRT"` | HTTPing，仅 HKG/NRT |

### 2.3 下载测速与输出

| 用例 ID | 参数 | 预期 |
|---------|------|------|
| OUT-01 | （无） | 启用下载测速，result.csv，10 条 |
| OUT-02 | `-dd` | 禁用下载测速 |
| OUT-03 | `-o out.csv -p 5` | 输出 out.csv，最多 5 条 |
| OUT-04 | `-silent` | 静默，仅输出 IP |
| OUT-05 | `-q -onlyip only.txt` | 同 -silent，输出到 only.txt |

### 2.4 定时与 Hosts

| 用例 ID | 参数 | 预期 |
|---------|------|------|
| SCH-01 | `-interval 60` | 每 60 分钟执行 |
| SCH-02 | `-at "6:00,18:00"` | 每日 6:00、18:00 |
| SCH-03 | `-cron "0 */6 * * *"` | Cron 每 6 小时 |
| SCH-04 | `-interval 1 -at "00:00"` | 多参数时按 cron > at > interval |
| HOST-01 | `-hosts "a.com"` | 测速后更新 hosts：a.com -> 第 1 名 IP |
| HOST-02 | `-hosts "1 b.com, 0 c.com"` | b.com 用第 2 名，c.com 用第 1 名 |
| HOST-03 | `-hosts "*.x.com" -hosts-dry-run` | 仅预览，不写入 |

### 2.5 其他与 API

| 用例 ID | 参数 | 预期 |
|---------|------|------|
| MISC-01 | `-allip` | 全量 IP（每 /24 不随机） |
| MISC-02 | `-debug` | 开启调试输出 |
| MISC-03 | `-api -api-port 9090` | 启用 API，端口 9090 |
| MISC-04 | `-useproxy` | 使用环境变量代理 |
| MISC-05 | `-useproxy http://127.0.0.1:7890` | 使用指定代理 |

## 3. 单元测试覆盖（代码级）

- **ConfigParser**：空参数、各开关、数值、-hosts 多次、-useproxy 有/无 URL、冲突调度参数。
- **IpProvider.ParseCidr**：合法 IPv4/IPv6 CIDR、单 IP、非法格式；allIp true/false。
- **Scheduler.GetMode**：无调度、interval、at、cron 及组合优先级。
- **HostsUpdater**：ParseHostsWithIndex（"0 a.com", "1 b.com", "*.c.com"）。
- **ColoProvider**：GetColoNameZh、IsColoAllowed、ParseCfColo。

## 4. 集成场景（需环境）

| 场景 | 命令示例 | 预期 |
|------|----------|------|
| 默认 IPv4 测速 | `cfst -ipn 10 -dd` | 需 ip.txt，仅延迟，最多 10 IP |
| IPv6 测速 | `cfst -ipv6 -ipn 10 -dd` | 需 ipv6.txt |
| -ip 指定 | `cfst -ip "104.16.0.0/24" -dd -n 5` | 不依赖文件，测该段 |
| 静默 | `cfst -ip "104.16.0.0/24" -dd -silent -ipn 5` | 退出码 0，仅输出 IP |
| 清理 | `.\build.ps1 -clean` | 删除 obj、bin、publish |

## 5. 运行测试

```bash
# 单元测试
dotnet test

# 指定项目
dotnet test tests/CloudflareST.Tests/CloudflareST.Tests.csproj

# 带日志
dotnet test --logger "console;verbosity=detailed"
```

## 6. 边界与异常

| 用例 | 输入 | 预期 |
|------|------|------|
| 无 IP 源 | 无 -ip、无 ip.txt/ipv6.txt | 抛 InvalidOperationException |
| -ip 非法 | `-ip "not-cidr"` | LoadAsync 中 ParseCidr 抛 FormatException；单行解析失败会 catch 后 TryParse 单 IP |
| -p 0 | `-p 0` | 按文档视为 10（SpeedTestRunner 中 limit = OutputNum <= 0 ? 10 : OutputNum） |
| 调度全无 | 无 -interval/-at/-cron | ScheduleMode.None |
| IPv4 prefix > 32 | 如 192.168.1.0/99 | ParseCidr 不抛，hostBits<=0 时仅返回 baseIp |
