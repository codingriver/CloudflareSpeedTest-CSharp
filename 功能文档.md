# Cloudflare Speed Test C# - 功能文档

> 基于项目源码分析的功能说明文档

---

## 1. 项目概述

**Cloudflare Speed Test** 是一个从 Cloudflare CDN 众多 IP 中测出**延迟低、速度快**节点的工具，基于 [XIU2/CloudflareSpeedTest](https://github.com/XIU2/CloudflareSpeedTest) 的 C# 实现。

### 1.1 核心价值

- 帮助用户从 Cloudflare 海量 IP 中筛选最优节点
- 支持自建优选、代理配置、Hosts 优化等场景
- 跨平台（Windows / Linux / macOS），单文件运行

### 1.2 技术栈

| 技术 | 说明 |
|------|------|
| 语言 | C# (.NET 8.0) |
| 依赖 | Cronos（Cron 表达式解析） |
| 运行方式 | 单文件可执行，无需安装 .NET |

## 3. 核心功能详解

### 3.1 延迟测试（三种模式）

| 模式 | 类 | 说明 | 适用场景 |
|------|-----|------|----------|
| **ICMP Ping** | IcmpPinger | 系统 Ping，需 ICMP 权限 | 默认，延迟最准确 |
| **TCPing** | PingTester | TCP 443 连接测时 | 网络禁用 ICMP 时 |
| **HTTPing** | HttpingTester | HTTP 请求测延迟 | 需地区码、可 -cfcolo 过滤 |

**流程**：加载 IP → 并发延迟测试 → 过滤（延迟/丢包率）→ 达标 IP 进入下载测速

### 3.2 下载测速

- **实现**：`SpeedTester` 通过 `ConnectCallback` 将 HTTP 请求绑定到待测 IP
- **测速 URL**：默认 `https://speed.cloudflare.com/__down?bytes=52428800`
- **排序**：有速度的按速度降序，无速度的按延迟顺序排在后面
- **代理**：支持 `-useproxy` 使用环境变量代理，或 `-useproxy http://...` 指定代理

### 3.3 地区码解析

- **实现**：`ColoProvider` 从 HTTP 响应头解析
- **支持 CDN**：Cloudflare、AWS CloudFront、Fastly、CDN77、BunnyCDN、Gcore
- **响应头**：`cf-ray`、`x-amz-cf-pop`、`x-served-by`、`x-77-pop`、`x-id-fe` 等
- **HTTPing 专属**：`-cfcolo "HKG,NRT,LAX"` 仅保留指定地区

### 3.4 IP 列表加载

- **来源**：本地文件（ip.txt/ipv6.txt）、`-ip` 参数、自动下载
- **自动下载**：多源回退（jsDelivr → GitHub Raw → ghproxy → mirror.ghproxy）
- **采样**：默认每 /24 随机一个 IP，`-allip` 时全量
- **IPv4/IPv6**：默认仅加载 ip.txt（IPv4），`-ipv6` 时仅加载 ipv6.txt（IPv6）。指定 `-ip` 或 `-f`/`-f6` 文件时，不按类型过滤，解析到啥就测啥。

### 3.5 Hosts 自动更新

- **格式**：`"N 域名1 域名2, M 域名3"`，N/M 为测速结果排名（0=最快）
- **通配符**：`*.example.com` 匹配 www.example.com、api.example.com 等
- **权限**：Windows 需管理员，Linux/macOS 需 root/sudo；无权限时输出到 `hosts-pending.txt`
- **预览**：`-hosts-dry-run` 仅输出待写入内容

### 3.6 定时调度

| 模式 | 参数 | 说明 |
|------|------|------|
| Interval | `-interval N` | 每 N 分钟执行一次 |
| At | `-at "6:00,18:00"` | 每日定点 |
| Cron | `-cron "0 */6 * * *"` | Cron 表达式 |
| 时区 | `-tz "China Standard Time"` | 指定时区 |

**优先级**：Cron > At > Interval

### 3.7 API 服务器

- **启动**：`-api -api-port 8080`
- **端点**：`/api/speedtest`、`/api/results`、`/api/config`、`/api/stop`
- **用途**：供 GUI、脚本、其他进程调用

## 4. 数据流与配置

### 4.1 测速流程

```
IP 列表 (IpProvider)
    ↓
延迟测试 (IcmpPinger / PingTester / HttpingTester)
    ↓ 过滤：DelayThresholdMs, DelayMinMs, LossRateThreshold
延迟达标 IP
    ↓
下载测速 (SpeedTester) [可选，-dd 禁用]
    ↓ 排序：速度降序 + 未测速按原序
最终结果
    ↓
输出 (OutputWriter) + Hosts 更新 (HostsUpdater)
```

### 4.2 配置类 (Config) 主要字段

| 分类 | 字段 | 默认值 | 说明 |
|------|------|--------|------|
| IP | IpFile, IpFileV6 | ip.txt, ipv6.txt | 段文件路径 |
| IP | IpRanges, MaxIpCount | null, 0 | 直接指定 CIDR，数量上限 |
| 延迟 | PingThreads, PingCount | 200, 4 | 并发数、每 IP 测速次数 |
| 延迟 | DelayThresholdMs, DelayMinMs | 9999, 0 | 延迟上下限 |
| 延迟 | LossRateThreshold | 1.0 | 丢包率上限 |
| 模式 | TcpPingMode, HttpingMode | false | 测速模式 |
| 下载 | SpeedUrl, SpeedNum | Cloudflare 官方, 10 | 测速 URL、参与数 |
| 下载 | DisableSpeedTest | false | 禁用下载测速 |
| 输出 | OutputFile, OutputNum | result.csv, 10 | CSV 路径、输出数量 |
| 调度 | IntervalMinutes, AtTimes, CronExpression | 0, null, null | 定时参数 |
| Hosts | HostsDomains, HostsDryRun | null, false | 域名、预览模式 |
| API | EnableApi, ApiPort | false, 8080 | API 开关、端口 |
| 代理 | UseProxy, ProxyUrl | false, null | 代理开关、URL |

## 5. 输出说明

### 5.1 控制台表格

```
序号    IP 地址            丢包率    延迟          抖动      下载速度      地区码    地区
--------------------------------------------------------------------------------------------------------
1       104.19.55.123      0%        50 ms        2.1 ms    82.63 Mbps   HKG      香港
```

### 5.2 CSV 格式

```csv
IP,丢包率,平均延迟(ms),最小延迟(ms),最大延迟(ms),抖动(ms),下载速度(Mbps),地区码,地区
104.19.55.123,0.00%,50,48,52,2.1,82.63,HKG,香港
```

### 5.3 静默模式 (-silent / -q)

- 仅输出 IP，每行一个
- 同时写入 `onlyip.txt`（可用 `-onlyip` 指定路径）
- 出错或 0 结果时写空文件
