## 1. 项目概述

### 1.1 什么是 Cloudflare Speed Test？

**Cloudflare Speed Test** 是一个从 Cloudflare CDN 众多 IP 中帮你测出**延迟低、速度快**节点的工具。

### 1.2 核心特性


| 特性           | 说明                                         |
| ------------ | ------------------------------------------ |
| **三种测速方式**   | ICMP Ping（默认）、TCPing、HTTPing               |
| **下载测速**     | 测延迟后还会测下载速度，结果更直观                          |
| **地区码解析**    | 支持 Cloudflare、AWS、Fastly、CDN77、Bunny、Gcore |
| **定时调度**     | 支持间隔、每日定点、Cron 表达式                         |
| **Hosts 更新** | 自动更新系统 hosts 文件                            |
| **跨平台**      | Windows / Linux / macOS                    |


### 1.3 技术栈

- **语言**: C# (.NET 8.0)
- **依赖**: Cronos（定时任务）
- **无需安装 .NET**：提供预编译的可执行文件

## 2. 核心功能

### 2.1 延迟测试

测试每个 IP 的响应延迟，支持：

- **ICMP Ping**（默认）：需要系统允许 Ping
- **TCPing**：走 TCP 443 端口，无需 ICMP 权限
- **HTTPing**：使用 HTTP 请求测延迟，可获取地区码

### 2.2 下载测速

对延迟达标的 IP 进行下载测速，测量实际下载速度。

### 2.3 地区码解析

从 HTTP 响应头中解析 CDN 节点所在地区：

- Cloudflare: `cf-ray`
- AWS: `x-amz-cf-pop`
- Fastly: `x-served-by`
- CDN77: `x-77-pop`
- BunnyCDN: Server 头
- Gcore: `x-id-fe`

### 2.4 Hosts 自动更新

自动将测速结果写入系统 hosts 文件，支持：

- 精确域名匹配
- 通配符域名（`*.example.com`）
- 多域名同时更新
- 使用第 N 名 IP

### 2.5 定时调度

支持三种调度模式：

- **Interval**：每 N 分钟执行一次
- **At**：每日定点执行
- **Cron**：Cron 表达式定时执行

#### SpeedTestRunner.cs - 测速任务运行器

核心测速流程：

```csharp
public static class SpeedTestRunner
{
    public static async Task<IReadOnlyList<IPInfo>?> RunAsync(Config config, CancellationToken ct)
    {
        // 1. 加载 IP 列表
        var ips = await IpProvider.LoadAsync(config, ct);
        
        // 2. 延迟测试（根据模式选择）
        IReadOnlyList<IPInfo> delayResults;
        if (config.HttpingMode)
            delayResults = await HttpingTester.RunHttpingAsync(ips, config, ...);
        else if (config.TcpPingMode)
            delayResults = await PingTester.RunTcpPingAsync(ips, config, ...);
        else
            delayResults = await IcmpPinger.RunIcmpPingAsync(ips, config, ...);
        
        // 3. 下载测速（可选）
        if (!config.DisableSpeedTest)
            finalResults = await SpeedTester.RunDownloadSpeedAsync(delayResults, config, ...);
        
        // 4. 输出结果
        WriteResults(config, outputResults);
        
        // 5. 更新 Hosts
        UpdateHosts(config, finalResults);
    }
}
```

#### IpProvider.cs - IP 列表加载

支持三种来源：

1. **本地文件**：`ip.txt`、`ipv6.txt`
2. **命令行参数**：`-ip "173.245.48.0/20,104.16.0.0/13"`
3. **自动下载**：首次运行自动从 CDN 下载

下载源（多源回退）：

- jsDelivr CDN
- GitHub Raw
- ghproxy 代理
- mirror.ghproxy 代理

#### HostsUpdater.cs - Hosts 文件更新

支持格式：

```bash
# 单域名，使用第1名（最快）IP
-hosts "example.com"

# 多域名
-hosts "a.com,b.com,c.com"

# 使用第 N 名 IP
-hosts "0 example.com"    # 第1名
-hosts "1 example.com"    # 第2名

# 通配符域名
-hosts "*.example.com"

# 混合格式
-hosts "0 a.com, 1 b.com, *.c.com"
```

#### Windows


首次运行会：

1. 自动下载 Cloudflare IP 列表到 `ip.txt`
2. 开始延迟测试
3. 对延迟达标的 IP 进行下载测速
4. 输出结果到控制台和 `result.csv`

```bash
# ICMP Ping（默认，需要 ICMP 权限）

./cfst -httping
```

## 5. 测速模式详解

### 5.1 ICMP Ping（默认）

- 使用系统 `Ping` 类进行测速
- 需要管理员/root 权限（Windows）
- 无需额外权限（Linux/macOS）
- **优点**：延迟最准确
- **缺点**：部分网络禁用了 ICMP

### 5.2 TCPing

- 尝试建立 TCP 连接到 443 端口
- **无需 ICMP 权限**
- **优点**：网络受限环境也能用
- **缺点**：延迟略高于 ICMP

### 5.3 HTTPing

- 发送 HTTP 请求测延迟
- 可以从响应头获取地区码
- 可用 `-cfcolo` 过滤特定地区
- **优点**：可筛选地区
- **缺点**：需要 HTTP 端口可访问

## 7. 定时任务与自动化

### 7.1 间隔模式

每 6 小时执行一次：

```bash
./cfst -interval 360 -silent -dd
```

### 7.2 每日定点

每天 6:00 和 18:00 执行：

```bash
./cfst -at "6:00,18:00" -silent
```

### 7.3 Cron 表达式

每 6 小时整点执行：

```bash
./cfst -cron "0 */6 * * *" -silent -dd
```

每周一 0:00 执行：

```bash
./cfst -cron "0 0 * * 1" -silent
```

### 7.4 时区设置

```bash
# 使用 UTC 时区
./cfst -cron "0 */6 * * *" -tz UTC

# 使用北京时间
./cfst -at "6:00,18:00" -tz "China Standard Time"
```

### 7.5 定时任务 + Hosts 更新

TCPing 测速，每 12 小时执行，更新 hosts：

```bash
./cfst -tcping -interval 720 -hosts "a.b.com" -silent
```

HTTPing 测速，每天早上 6 点执行，更新 hosts：

```bash
./cfst -httping -at "6:00" -hosts "a.b.com,c.b.com" -silent
```

## 8. Hosts 自动更新

### 8.1 基本用法

```bash
# 测速后更新 hosts
./cfst -hosts "cdn.example.com" -silent

# 使用第 2 名 IP
./cfst -hosts "example.com" -hosts-ip 2 -silent
```

### 8.2 多域名

```bash
./cfst -hosts "cdn.example.com,api.example.com,*.example.com" -silent
```

```bash
# 更新匹配 *.example.com 的域名
./cfst -hosts "*.example.com" -silent
```

```bash
# 0 = 第1名（最快），1 = 第2名，以此类推
./cfst -hosts "0 a.com, 1 b.com" -silent
```

- **Linux/macOS**：需要 root 权限或 sudo

### 8.6 预览模式

```bash
# 只预览要写入的内容，不实际写入

```

## 11. 常见问题与解决方案

### 11.1 没有 IP 列表

**问题**：提示 "没有可测速的 IP"

**解决**：

```bash
# 首次运行会自动下载
./cfst

# 手动下载
curl -o ip.txt https://cdn.jsdelivr.net/gh/codingriver/CloudflareIP-Sync@main/ip.txt
```

### 11.2 ICMP 测不出结果

**问题**：ICMP Ping 全部超时

**解决**：改用 TCPing 或 HTTPing

```bash
./cfst -tcping
# 或
./cfst -httping
```

### 11.3 权限不足

**问题**：更新 hosts 提示无权限

**解决**：

- Windows：以管理员身份运行
- Linux/macOS：使用 sudo
- 或查看 `hosts-pending.txt` 手动合并

### 11.4 只想测延迟

**问题**：不需要下载测速

**解决**：

```bash
./cfst -dd          # 禁用下载测速
./cfst -dd -tcping # TCPing + 只测延迟
```

### 11.5 测速太慢

**问题**：IP 太多，测不完

**解决**：

```bash
# 限制 IP 数量
./cfst -ipn 500

# 限制并发
./cfst -n 500 -dn 20 -t 8
```

### 11.6 换测速地址

**问题**：默认地址太慢或不可用

**解决**：

```bash
# 使用其他测速地址
./cfst -url "http://speedtest.303066.xyz/__down?bytes=104857600" -tp 80
./cfst -url "https://speed.cloudflare.com/__down?bytes=52428800"
```

### 11.7 筛选特定地区

**问题**：只想测特定地区的节点

**解决**（仅 HTTPing）：

```bash
./cfst -httping -cfcolo "HKG,NRT,LAX"
```

```bash
./cfst -sl 40
```

### 12.5 延迟阈值过滤

```bash
# 延迟 <= 100ms
./cfst -tl 100

# 延迟 50-150ms
./cfst -tl 150 -tll 50
```

### 12.6 输出更多结果

```bash
# 输出 20 个结果
./cfst -p 20

# 输出到指定文件
./cfst -o my-result.csv
```

### 12.7 调试模式

```bash
# 显示详细调试信息
./cfst -debug
```

### 12.8 API 服务器

```bash
# 启动 API 服务器
./cfst -api -api-port 8080

# 定时测速 + API
./cfst -interval 60 -api
```
